## Code of Conduct

This project adheres to ethical security research principles:

- **Responsible Disclosure**: Report vulnerabilities responsibly
- **Educational Purpose**: All contributions should serve educational goals
- **No Malicious Use**: Do not use this research for cheating or illegal activities
- **Respect Privacy**: Do not collect or share personal information

## How to Contribute

### Areas for Contribution

1. **Documentation Improvements**
   - Fix typos or clarify explanations
   - Add examples or use cases
   - Improve code comments

2. **Testing Enhancements**
   - Add new test cases
   - Improve test coverage
   - Add automated testing

3. **Security Analysis**
   - Identify additional vulnerabilities
   - Analyze countermeasures
   - Document attack vectors

4. **Code Improvements**
   - Optimize bypass techniques
   - Add error handling
   - Improve code structure

---

## Contribution Process

### 1. Fork the Repository

1. Fork this repository to your GitHub account
2. Clone your fork locally
3. Create a new branch for your contribution

### 2. Make Your Changes

1. Make your changes in the new branch
2. Test your changes thoroughly
3. Update documentation if needed
4. Follow code style and formatting

### 3. Submit a Pull Request

1. Push your changes to your fork
2. Create a pull request with:
   - Clear description of changes
   - Reason for the change
   - Testing performed
   - Any breaking changes

### 4. Review Process

- All pull requests will be reviewed
- Feedback may be requested
- Changes may be requested before merging

## Coding Standards

### JavaScript Style

- Use meaningful variable names
- Add comments for complex logic
- Follow existing code structure
- Maintain consistency with existing code

### Documentation

- Update relevant documentation files
- Add author credits to new files
- Include code examples
- Explain "why" not just "what"

### Testing

- Test all changes thoroughly
- Add tests for new features
- Verify backward compatibility
- Document test procedures

## Security Research Guidelines

### Responsible Disclosure

If you discover new vulnerabilities:

1. **Do not** create public issues
2. Contact the maintainer directly via email
3. Provide detailed information
4. Allow reasonable time for response

### Proof of Concept

When submitting bypass techniques:

1. Include working code
2. Explain the vulnerability
3. Document the attack vector
4. Suggest countermeasures

---

## Pull Request Guidelines

### Required Information

- **Title**: Clear, descriptive title
- **Description**: Detailed explanation of changes
- **Testing**: How changes were tested
- **Impact**: What this changes or improves

### Example Pull Request

```markdown
## Title
Add keyboard shortcut test for F1-F12 keys

## Description
Extends keyboard testing to include function keys (F1-F12) which
are sometimes blocked by proctoring systems.

## Changes
- Added F1-F12 key tests in testing-guide.md
- Updated testAllBypasses() to include function keys
- Added documentation for function key bypass

## Testing
- Tested on Chrome and Edge
- Verified F1-F12 keys work after bypass
- Confirmed tests pass

## Impact
Improves test coverage and documentation
```
